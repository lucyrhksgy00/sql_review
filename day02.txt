SQL(Structured Query Language)
	데이터베이스관리시스템(DBMS)의 데이터를 관리하기 위해 설계된 언어
	SQL문, 쿼리, 질의문

SQL문의 주요 특징
	한 줄 한 줄씩 실행되는 인터프리터 언어
	대문자와 소문자는 구분하지 않는다
		1. 예약어(이미 역할이 정해져있는 단어)는 대문자로,
		   나머지는 소문자로 작성한다(가독성을 높이기 위해서)
		2. 너무 길어지면 가독성을 높이기 위해서 줄바꿈을 할 수 있다
		3. 들여쓰기를 사용하여 가독성을 높일 수 있다

관계형데이터베이스관리시스템(RDBMS)
	2차원의 표 형태의 테이블이 관계를 맺고있다

	테이블 : object(객체) / entity(개체), 데이터가 저장되는 공간, 행과 열로 이루어져있다
	행 : row, 가로줄, 레코드, instance, 튜플
	열 : column, 세로줄, 속성
	행과 열이 만나는 교점 : 필드

조회
SELECT 문
	SELECT 컬럼명, 컬럼명, ... 
	FROM 테이블명
	ORDER BY 컬럼명 [ASC or DESC];

값의 종류
	숫자		NUMBER			10, -5.1, ...
	문자(열)	VARCHAR2 / CHAR		'안녕하세요', 'steven', ...
	날짜		DATE				년월일시간분초
	
	'10' 문자타입의 값
	
	하나의 컬럼에는 같은 타입의 값만 들어갈 수 있다

연산자
	연결연산자
		a || b --> ab
			두 값을 연결하여 하나의 값으로 만들어준다
	산술연산자
		+ - * /
		DATE 타입의 값인 경우에는 + - 사용 가능, 일이 계산된다
	관계(비교)연산자
		a > b		: a가 b보다 큰 조건
		a < b		: a가 b보다 작은 조건
		a >= b	: a가 b보다 크거나 같은 조건
		a <= b	: a가 b보다 작거나 같은 조건
		a = b		: a가 b랑 같은 조건
		a <> b	: a랑 b가 다른 조건
	SQL연산자
		a IN (b, c, ...) : a가 b 혹은 c 혹은 ...인 조건
		a BETWEEN b AND c : a가 b 이상 c 이하인 조건
		a LIKE 패턴 : a가 패턴을 만족하는 조건
			% : ~로 해석
			_ : 자릿수로 해석
	논리연산자
		a AND b : a 조건도 만족하고 그리고 b 조건도 만족하는 조건
		a OR b  : a 조건 혹은 b 조건 둘 중 하나라도 만족하는 조건
		NOT a   : a 조건을 만족하지 않는 조건
